### 第一天：只看极简手册，建立“黑盒”概念

**不要去看几百页的RISC-V全量手册！不要去看各种复杂的论文！**

1.  **下载《RISC-V指令集手册（非特权级）》：** 网上有很多中文翻译版。
2.  **只看 RV32I 基础整数指令集：** 这一章大概只有十几页。
3.  **你要搞懂什么：**
    *   **寄存器：** RISC-V有32个通用寄存器（`x0`到`x31`），其中`x0`永远是0。
    *   **指令格式：** 看看R型、I型、S型、B型指令长什么样。知道每一条32位的指令，哪几位是操作码（opcode），哪几位是源寄存器（rs1, rs2），哪几位是目标寄存器（rd）。
    *   *目标：* 此时CPU对你来说是个黑盒，进去的是32位的0101机器码，出来的是寄存器里数值的变化。这就够了。

### 第二天：让代码在你的电脑上“动起来”（最关键的一步）

**不要直接去读那个GitHub项目的长篇Verilog代码，会看晕的！先让它跑起来！**

1.  **打开仿真器（ModelSim 或 Vivado自带的Simulator）：** 把那个GitHub项目里关于CPU核心的 `.v` 文件全加进去。
2.  **寻找或编写 Testbench（测试文件）：** 顶层一定有一个 `tb.v` 或者 `cpu_tb.v`。这个文件的作用就是给CPU提供一个时钟（clk）和复位信号（rst）。
3.  **加载一段极其简单的机器码：**
    *   不用C语言，直接手写两行汇编对应的十六进制机器码。
    *   比如：
        `addi x1, x0, 5`  (把5存入x1寄存器)
        `addi x2, x0, 10` (把10存入x2寄存器)
        `add x3, x1, x2`  (x1+x2，结果15存入x3)
    *   把它们转成十六进制，放进一个 `inst.txt` 文件里，用 `$readmemh` 读进仿真器的ROM里。
4.  **看波形，找成就感！**
    *   运行仿真。把 PC（程序计数器）拉出来，把 通用寄存器堆（RegFile）里的 `x1`、`x2`、`x3` 拉出来。
    *   当你在波形图上看到 PC 从 0 变成 4，变成 8... 然后看到 `x3` 的值在某一个时钟沿真的变成了 15（十六进制的 `0xf`）。**恭喜你，你已经驾驭了一个 CPU！恐惧感会瞬间消失一半。**

### 第三天：“顺藤摸瓜”看代码（追踪一条指令的一生）

现在你开始有底气了，我们来“解剖”这个5级流水线。**不要泛泛地看，要“顺藤摸瓜”地看。**

还是用刚才那条 `add x3, x1, x2` 指令，你在代码里搜寻它的足迹：

1.  **取指 (IF)：** 找到 `PC` 寄存器在哪，找它是怎么去内存里把那32位指令拿出来的。
2.  **译码 (ID)：** 找到一个叫 `Decoder` 或 `Ctrl` 的模块。看它是怎么用 `assign opcode = inst[6:0];` 这种切片代码，把指令拆开的。看它是怎么识别出这是一条 `add` 指令，并把控制信号（比如 `ALUOp`）拉高的。
3.  **执行 (EX)：** 找到 `ALU` 模块。看它是不是接收了 `x1` 和 `x2` 的值，以及加法控制信号，然后在内部写了一个简单的 `assign out = in1 + in2;`。
4.  **访存 (MEM)：** 因为是 `add` 指令，不需要读写内存。看看数据是怎么“打酱油”穿过这个阶段的。
5.  **写回 (WB)：** 看看最后的结果 15，是怎么顺着线，连回 Register File 的写端口的，同时写使能信号 `RegWrite` 是不是为 1。

*   *目标：看完这一圈，你就在脑子里把《计组》课本上的数据通路图，和 Verilog 代码完全对应上了！*

### 第四天：直面大Boss——流水线寄存器与数据冲突

这是5级流水线最核心的难点，也是你作为“同学A”必须搞懂并向评委吹嘘的资本。

1.  **找流水线寄存器：** 寻找名字叫 `IF_ID.v`, `ID_EX.v`, `EX_MEM.v`, `MEM_WB.v` 的模块（或者在一个大文件里用 `always @(posedge clk)` 写的打拍逻辑）。它们就是阻隔这5个阶段的“水闸”。
2.  **寻找旁路/前推单元 (Forwarding Unit)：**
    *   在代码里搜索 `Forward` 或者 `Bypass` 字样。
    *   看看代码里是不是有类似这样的判断：
        `if (EX_MEM_RegWrite && (EX_MEM_Rd != 0) && (EX_MEM_Rd == ID_EX_Rs1))`
    *   **对照课本理解这句话的意思：** “如果上一条指令要写寄存器，且写的不是0号寄存器，且写的目标刚好是当前指令需要的源操作数，那么就直接把上一条指令的结果通过MUX短接过来！”
    *   *懂了这个，你的5级流水线就通关了80%。*

### 第五天：剥离与净化（为你队友B的SoC集成做准备）

那个GitHub项目可能带了一些乱七八糟的Cache或者它自己写的奇怪内存模拟。你现在要把它“净化”成一个纯粹的 CPU Core。

1.  **砍掉多余枝节：** 把关于 UART、Cache 的代码注释掉。
2.  **整理接口：** 让你的 CPU Core 模块看起来像这样干干净净：
    ```verilog
    module cpu_core (
        input  clk,
        input  rst_n,
        // --- 指令内存接口 (连向同学B的ROM) ---
        output [31:0] inst_addr,
        input  [31:0] inst_data,
        // --- 数据内存接口 (连向同学B的RAM或总线) ---
        output        mem_we,       // 写使能
        output [31:0] mem_addr,     // 读写地址
        output[31:0] mem_wdata,    // 要写的数据
        input  [31:0] mem_rdata     // 读出的数据
    );
    ```
3.  **交接：** 把这个干干净净的 `cpu_core.v` （以及它内部的 IF/ID/ALU 等子模块）打包发给**同学B**，告诉他：“CPU心脏我剥离出来了，且用测试波形验证无误了！你快去给它套上总线，连上外设和加速器吧！”

### 👨‍💻 给 同学B 的作战指南：SoC集成与“秘密武器”打造
**你的核心任务：** 接收A的裸CPU，给它套上总线，连上内存和外设，并**打造本次比赛的核心得分点——硬件加速器**。

#### 第一周：搭建“空壳”SoC与内存映射 (Memory Map)
1.  **定义系统地盘（极其重要）：** 和C一起商量好物理地址怎么分配。
    *   例如：`0x0000_0000`~`0x0000_FFFF` (64KB) 给指令ROM；`0x2000_0000`~`0x2000_FFFF` (64KB) 给数据RAM。
    *   例如：`0x4000_0000` 给UART串口，`0x5000_0000` 给你们的加速器。
    *   **把这个表格画得漂漂亮亮，这是初赛报告的必考项！**
2.  **手搓简易总线（Bus Interconnect）：**
    *   不要去死磕完整的 AXI 或 AHB 协议，时间不够！
    *   **写一个简单的地址译码器（Address Decoder）：** 用一个 `case` 语句或者 `if-else`，根据 A 传出来的 `Mem_Addr`，把读写使能信号分发给对应的 ROM、RAM 或 外设。
3.  **连接内存：** 用 Vivado 的 IP 核生成两块 BRAM，一块做指令存储，一块做数据存储。连到你的简易总线上。

#### 第二周：找开源外设，点亮UART
1.  **拒绝手写外设：** 去 Github 找一个极简的开源 UART（带 APB 接口或者直接裸接口的都行）。
2.  **挂载UART：** 把 UART 挂到你的总线上，分配好地址。它的 `TX` 引脚接到顶层模块输出。
3.  **系统合体：** 把同学A交给你的 `cpu_core.v` 实例化到你的 SoC 顶层（`soc_top.v`）里。此时，CPU 已经可以通过总线摸到内存和 UART 了。

#### 第三周：打造“秘密武器”——硬件加速器（拿高分全靠它！）
1.  **看赛题选算法：** 选一个适合硬件做的算法（比如：矩阵乘法、FIR滤波、简单的AES/SM4加密的一个轮函数）。
2.  **设计加速器IP：** 用 Verilog 写这个算法模块。它不需要很复杂，只要能体现出**“比软件CPU算得快”**就行。
3.  **挂载加速器：** 把这个加速器也挂到你的总线上，给它分配几个控制寄存器（比如：地址0写数据A，地址4写数据B，地址8写控制字启动运算，地址12读出结果）。这就是经典的 **MMIO（内存映射I/O）** 设计。

#### 第四周：配合C联调，画精美架构图
*   在仿真软件里，配合 C 抓取 CPU 通过总线读写加速器的波形图。
*   用 Visio 画出超高逼格的 **SoC 顶层架构图**（包含 CPU、Bus Matrix、SRAM、UART、Accelerator），放进初赛报告。

### 👨‍💻 给 同学C 的作战指南：工具链、驱动与“上帝视角”验证
**你的核心任务：** 搞定把C语言变成机器码的工具链，写出让硬件动起来的代码，并**收集初赛报告最需要的“定海神针”——带注释的仿真波形图**。

#### 第一周：打通软件工具链（最容易让人崩溃的一周）
1.  **装环境：** 下载并安装 `riscv-none-elf-gcc` 交叉编译工具链（Windows下推荐用芯来科技的预编译包，或者 xPack RISC-V gcc）。
2.  **抄底层启动代码（Boot/Linker）：** 裸机C语言是不能直接运行的，需要一段汇编代码（比如 `start.S`）来初始化栈指针（SP），然后跳转到 `main` 函数。同时需要一个链接脚本（`link.ld`）来规定代码放在哪个地址。
    *   **绝招：** 直接去 Github 上的 `tinyriscv` 或 `picorv32` 项目里，把他们的 `start.S` 和 `link.ld` 复制过来，改成你们和 B 商量好的地址范围。
3.  **写出第一段机器码：** 写一个只做加减法的 `main.c`，写一个 `Makefile`，敲一下 `make`，能成功生成 `.hex` 或 `.bin` 文件，你的第一周任务就完美完成了！

#### 第二周：编写底层驱动（BSP）
1.  **用指针操控硬件：** 根据 B 给你的内存映射表，用 C 语言宏定义写外设的地址。
    *   例如：`#define UART_TX_REG (*(volatile unsigned int *)0x40000000)`
2.  **重定向 printf：** 写一个把字符丢进 `UART_TX_REG` 的函数。这样你就可以在 C 语言里直接用 `printf` 打印信息了。这在后面的联合调试中是神级技能！
3.  **开始联合仿真：** 把你生成的 `.hex` 文件发给 A 和 B。他们放进 ModelSim 仿真，你应该能在 ModelSim 的 Transcript 窗口看到你的 `printf` 输出（这需要B在测试平台 `tb.v` 里写一点接收UART的简单逻辑）。

#### 第三周：编写软硬件协同应用程序（验证加速器）
1.  **写加速器驱动：** B 把加速器做好了。你现在要写 C 代码测试它。
    *   先用纯 C 语言（软件）算一遍矩阵乘法，记录下周期数（利用RISC-V的指令去读计数器，或者看波形时间）。
    *   再用 C 语言往 B 设计的加速器地址里写配置，让硬件加速器算一遍矩阵乘法。
2.  **证明你们的价值：** 在报告里对比：“纯软件算需要 1000 个周期，用我们设计的硬件加速器只要 20 个周期，性能提升了50倍！”——**评委最喜欢看这个！**

#### 第四周：收集波形，主笔测试报告
*   初赛是不需要上板演示的，评委怎么知道你们做出来了？**全靠带批注的波形图！**
*   你在 ModelSim 里跑你们最终的应用程序。
*   把以下波形截图，用红框圈出来，并在报告里配字：
    1.  “图X：这是软件向加速器配置寄存器写入数据的总线时序...”
    2.  “图Y：这是加速器计算完成，CPU通过Load指令读回正确结果的时序...”
    3.  “图Z：这是UART通过TX引脚输出 'Success' 字符串的波形...”
